# 각 탭 인라인 데이터 입력 + [object Object] 이메일 수정

## 수정 대상

### 1. 견적서 탭 — "새 견적서" 버튼 추가
견적서는 복잡한 위저드 폼이므로 인라인 생성 대신 견적서 생성 페이지로 이동하는 버튼 추가.
고객사 정보(customerId, company)를 쿼리 파라미터로 전달.

### 2. 계약 탭 — "새 계약" 버튼 추가
계약도 복잡한 폼이므로 계약 생성 페이지로 이동하는 버튼 추가.
고객사 정보를 쿼리 파라미터로 전달.

### 3. 시험접수 탭 — 인라인 생성 폼 추가
간단한 인라인 폼으로 시험접수 직접 생성 가능하게.

### 4. [object Object] 이메일 버그
헤더 영역에서 customer.email이 object인 경우 방어 처리 (이전 수정이 적용 안 된 부분 확인).

## 연동 원칙
모든 탭에서 생성된 데이터는 해당 고객의 customerId로 연결되므로
다른 모듈에서도 동일하게 조회됨 (findRelatedCustomerIds 크로스 조회 적용).
